\begin{abstract}\noindent
\small
\textbf{AI News Monitor} is a production-grade MLOps pipeline for four-class news classification (World, Sports, Business, Technology). It packages the full ML lifecycle---ingestion, feature engineering, training, experiment tracking, registry, containerized serving, observability, and continuous deployment---behind a self-healing loop that detects drift, retrains a DistilBERT classifier on user-corrected labels, and promotes improved models only when a statistical gate is met. A Thompson-Sampling A/B router over Beta posteriors evaluates challengers online without position bias. The stack is reproducible via Docker Compose (nine services), observable through MLflow, Prometheus, and Grafana, and deployable via GitHub Actions.
\end{abstract}

\vspace{-0.5em}

\section{Project Overview and Objectives}

The hard problem of modern ML is not launch-day accuracy but \emph{sustained} accuracy as the world drifts beneath a deployed model. News classifiers degrade quickly: vocabulary evolves, topic distributions shift, and feedback accumulates. AI News Monitor closes the loop---every step from data logging through retraining to promotion is automated, observable, and version-controlled.

\paragraph{Objectives.}
\begin{itemize}
    \item \textbf{Automated drift detection} on predicted label distribution (Pearson $\chi^2$) and confidence calibration (symmetric KL) against reference baselines.
    \item \textbf{Self-healing retraining} that pulls user-corrected labels and fine-tunes DistilBERT with Bayesian Beta-weighted cross-entropy.
    \item \textbf{Statistical promotion gate}: only promote if $\Delta F_1 \geq 2\%$; otherwise retain the incumbent.
    \item \textbf{Bayesian A/B routing} via Thompson Sampling over per-model Beta posteriors for unbiased online evaluation.
    \item \textbf{End-to-end observability} through Prometheus metrics, Grafana dashboards, and MLflow experiment tracking.
    \item \textbf{Reproducible infrastructure}: Docker Compose (nine services), Alembic migrations, GitHub Actions CI/CD, HuggingFace Hub.
\end{itemize}

\subsection{Drift Metrics}

\textbf{Label drift} is measured with Pearson's $\chi^2$ goodness-of-fit test over 4 class bins:
\[
\chi^2 = \sum_{i=1}^{4} \frac{(O_i - E_i)^2}{E_i}, \qquad p = 1 - F_{\chi^2_3}(\chi^2),
\]
where $O_i$ is the observed frequency in a rolling 500-prediction window and $E_i$ is the reference frequency from the validation set. Drift is declared when $p < 0.05$.

\textbf{Confidence drift} uses the symmetric Kullback--Leibler divergence between softmax-confidence histograms (10 bins):
\[
D_{KL}^{\text{sym}}(P \,\|\, Q) \;=\; \tfrac{1}{2}\sum_{i} P_i \log\!\tfrac{P_i}{Q_i} \;+\; \tfrac{1}{2}\sum_{i} Q_i \log\!\tfrac{Q_i}{P_i},
\]
with a Laplace smoothing constant of $10^{-6}$ to avoid division by zero on empty bins. Drift is declared when $D_{KL}^{\text{sym}} > 0.15$. Both thresholds were calibrated empirically through synthetic drift-injection experiments (class-label permutations and temperature-scaled softmax perturbations).

\subsection{Retraining Pipeline}

On drift trigger, the retraining orchestrator (\texttt{backend/app/services/retrainer.py}) runs a fire-and-forget async task guarded by a Redis lock (at most one active run per model): (1) join recent predictions with feedback into a preference dataset, upsampling disagreements $2\times$; (2) fine-tune DistilBERT (lr $2\!\times\!10^{-5}$, AdamW, 10\% warmup, 3 epochs, batch 32, max seq 128) with class-weighted cross-entropy and Bayesian disagreement weights; (3) evaluate F1-macro, accuracy, and per-class recall on a held-out split; (4) if $\Delta F_1 \geq 2\%$, push to HuggingFace Hub, register as \texttt{Production} in MLflow, and hot-swap weights, else archive; (5) log params, metrics, and a confusion-matrix artifact to MLflow and W\&B.

\subsection{Bayesian A/B Routing}

For each model $m \in \{A, B\}$ we maintain a Beta posterior $p(\theta_m \mid \text{feedback}) = \text{Beta}(\alpha_m^0 + s_m, \beta_m^0 + f_m)$, where $s_m, f_m$ are successes and failures (uniform prior $\alpha_m^0 = \beta_m^0 = 1$). At request time we draw $\theta_m \sim \text{Beta}(\alpha_m, \beta_m)$ and route to $\arg\max_m \theta_m$ (Thompson Sampling). This converges to the better model in expectation while preserving exploration, avoiding the position-bias of fixed 50/50 splits.

\section{Results}
\begin{itemize}
    \item \textbf{Baseline:} DistilBERT on AG News reaches $\sim$\textbf{92\% accuracy} / \textbf{0.91 F1-macro} after 3 epochs.
    \item \textbf{Drift latency:} injected label shift detected within \textbf{3 cycles ($\sim$90\,min)}; confidence drift in \textbf{2 cycles}.
    \item \textbf{Retraining cost:} $\sim$5\,min on a single T4-class GPU; observed F1 recovery of \textbf{+1.5\% to +4\%}.
    \item \textbf{Serving:} p95 latency $<100$\,ms; Redis cache hit ratio $\sim$\textbf{60\%} cold, $\sim$\textbf{80\%} warm.
    \item \textbf{Reliability:} \textbf{12 automated redeployments} with zero downtime (hot-swap weights, graceful restart).
    \item \textbf{A/B:} Beta posteriors converged after $\sim$200 feedback events; Thompson routing favored the winner $>$\textbf{85\%} of the time while retaining exploration.
\end{itemize}

\section{Challenges, Feedback Addressed, and Deployment}

\textbf{Challenges.} (i) \emph{Event-loop blocking}: synchronous HuggingFace calls stalled FastAPI under load; fixed by \texttt{asyncio.to\_thread}. (ii) \emph{Retraining concurrency}: overlapping drift cycles spawned parallel retrains until a Redis lease enforced mutual exclusion. (iii) \emph{Threshold calibration}: static $\chi^2$/KL bounds over-triggered in low-traffic windows; fixed with a $\geq$500-sample minimum guard.

\textbf{Feedback resolved.} The proposal review asked for \emph{specific model choice} and \emph{concrete drift detail}. Addressed by (1) \textbf{DistilBERT-base-uncased} (66M params) for both variants; (2) concrete drift tests ($\chi^2$ at $p<0.05$; symmetric KL at $D_{KL}>0.15$) over 500-sample rolling windows; (3) a fully automated retrain$\to$evaluate$\to$gate$\to$promote pipeline; and (4) added Bayesian A/B routing, Prometheus/Grafana observability, and GitHub Actions CI/CD.

\textbf{Deployment.} \texttt{docker compose up} starts nine services: Frontend (\texttt{:3000}), Backend (\texttt{:8000}), MLflow (\texttt{:5000}), Grafana (\texttt{:3001}), Prometheus (\texttt{:9090}), MinIO (\texttt{:9001}), Postgres, Redis, MinIO-init. Production targets: Supabase (Postgres), Railway (FastAPI + Next.js), HuggingFace Hub (model artifacts); MLflow/Grafana/MinIO have managed-service equivalents.

Datasets: AG News (120k/7.6k), 20 Newsgroups. Base model: DistilBERT (Sanh et al., 2019). Statistics: Pearson $\chi^2$, symmetric Kullback--Leibler divergence, Beta-Bernoulli Thompson Sampling.
